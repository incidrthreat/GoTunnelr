ssh

In this lesson we will cover the following:

  • SSH Server
  • SSH Client
  • Basic SSH Syntax (*nix)
  • Basic SSH Syntax (Windows using PuTTY)

SSH Server is native to most distributions of *nix and once configured, can be started either at boot or manually.  By default, SSH listens on port 22, but can be changed by the system administrator to be something other than port 22.  Up until the release of the Anniversary update to Windows 10, and enabling “Developer mode” on that release, SSH was NOT native to the Windows OS environment.  Having said that, there are 3rd party applications available for previous versions of Windows (Pre-Win10-14352) such as MobaSSH and OpenSSH to name a couple.

SSH Client also native to *nix and available in Windows 10 post Anniversary update.  Previous to that, PuTTY was the most commonly tool used to access an SSH Server from Windows OS.

*NIX OS Basic SSH syntax consists of invoking the ssh client and providing some parameters to traverse the network(s) and attach to the SSH Service. 

With the first two examples, we are using ssh and a target hostname or ip address on a *nix OS to make these connections. With these minimal commands, ssh presumes we want to connect to the ssh server on a remote host as the currently logged in user on our local host (user1) and will make the initial connection then prompt us for the password of the assumed user (user1).

user1@host:~# ssh target
or
user1@host:~# ssh 192.168.1.1

Now, if user1 is not the user we want to connect as or doesn't exist on the remote server, we need to explicitly tell ssh who we want to be on the remote/target system. With that, the syntax changes slightly with the added username in the syntax. In the example below, we connect to a target machine as user2 and you will be prompted for a password related to that specific user.

user1@host:~# ssh user2@target
or
user1@host:~# ssh user2@192.168.1.1

Windows OS Basic SSH syntax uses a 3rd party application called PuTTY and can be found here. The Windows examples accomplish the same task of connecting to an ssh service. When we envoke putty.exe we should specify the protocol to use, we do this with the -ssh switch due to the protocol agnostic usecase of PuTTY. Similar to SSH, PuTTY presumes we want to connect as the currently logged in user on our host machine (user1) and will make the initial connection then prompt us for the password of the assumed user (user1).

C:\users\user1\putty.exe -ssh target
or
C:\users\user1\putty.exe -ssh 192.168.1.1

Just as we did in the *Nix Basic Syntax, we can add an additional parameter if a different user is needed. You will be prompted for the password of user2 upon sending the below attempt.

C:\users\user1\putty.exe -ssh user2@target
or
C:\users\user1\putty.exe -ssh user2@192.168.1.1

With these basic SSH commands on both *nix and Windows OS resepectively, you now have an understanding of how to access a remote system using the Basic Secure Shell (SSH) Commands to connect to an SSH Server using an SSH Client. Next, we will be visiting a few specific arguments and parameters that will allow us to do more advanced techniques with SSH.
